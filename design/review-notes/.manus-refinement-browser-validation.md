# Metis refinement browser validation

## Gaps
The rebuilt `/gaps` screen now reads as a first-class clarification ledger. One dominant work area is visible, open and resolved states are clearly separated by stronger amber and green treatments, and the right rail stays secondary. Navigation active state is noticeably stronger than before. One small watchpoint remains: the repeated action row on every gap card adds some vertical repetition, although it no longer reads like a task manager.

## Brief
The refined `/brief` screen keeps the document dominant, and the support rail is materially tighter than before. Readiness pills have more contrast and the current-state rail is easier to parse on the dark background. Remaining issue: the full-mode header still contains a few stacked labels and toggles before the document begins, so the top band is improved but not yet as lean as the body of the artifact.

## Setup
The rebuilt `/setup` screen now has one clear intake block with secondary support condensed into a single right rail. The page feels less stacked and less box-heavy than before, and the readiness pills are more legible against the dark background. Remaining issue: the template card and reference list are now compact, but the lower half is still denser than the very clean top intake region.

## Sources
The refined `/sources` screen now behaves like a single evidence ledger with the section-posture rail clearly secondary. Tier and readiness states are much easier to distinguish, especially the stronger validation and conflict treatments. Remaining issue: long evidence notes still create a tall scroll, so future tightening could collapse some metadata on lower-priority source rows.

## Input
The refined `/input` screen now clearly centers one attributable record, with the linked-input list reading as secondary evidence rather than a competing grid of cards. Contrast is improved through the stronger confidence and readiness pills, and the copy stays operator-facing. Remaining issue: the primary metadata slab is still a bordered panel inside the dominant record, so there is a little residual inset layering.

## Compare
The refined `/compare` screen now reads as a delta ledger rather than a collection of equal-weight summaries. The current-state posture gets stronger emphasis, and the circulation rail is easier to scan because the status colors now separate improved, stable, and blocked states more clearly. Remaining issue: the top-left title still repeats the review context above the main change content, so the header could be shortened slightly later.

## Export
The refined `/export` screen now gives clear visual dominance to package selection, while the preview rail supports rather than competes. Readiness and validation states have stronger separation, and the selected package is more obvious at a glance. Remaining issue: the preview artifact remains visually strong enough that it still competes slightly with the package list on wide screens, though much less than before.
