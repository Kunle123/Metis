# Metis Next Design Pass Report

## Summary

This pass tightened Metis around its intended category: a **source-backed issue-briefing workspace** for corporate communications and corporate affairs teams. The work followed the required sequence of **Setup**, **Sources**, **Gaps**, **Brief**, and then a **browser realism pass**. Across the pass, the product was pushed away from narrated concept-prototype behavior and closer to a credible working internal tool, while keeping the **brief** as the center of gravity.

| Pass | Primary change | Metis alignment effect | Remaining risk |
|---|---|---|---|
| Setup | Rebuilt around a dominant intake form with clear required/optional separation | Reinforces disciplined issue intake under pressure | Shared top snapshot still adds some chrome before the form |
| Sources | Reframed as evidence control with stronger tier grouping and clearer card structure | Makes source strength and weakness legible without drifting into monitoring | Introductory hero copy could be tighter |
| Gaps | Rebuilt around unresolved questions, affected brief sections, suggested roles, and drafted prompts | Strengthens clarification support without becoming task software | Action rail could be even more compact |
| Brief | Added readiness, changes, blockers, and executive mode while preserving document dominance | Makes the brief operationally useful and circulation-aware | Shared shell summary band can still compete with the briefing surface |
| Browser realism pass | Checked Setup, Sources, Gaps, and Brief in-browser and verified clean type-check/build | Confirms the prototype behaves like a plausible product shell | Additional trimming of residual explanatory copy would improve realism further |

## 1. Setup pass

The Setup screen was redesigned so the **form is the dominant work area**. Required briefing inputs are now grouped under **Issue intake** and visually separated from **Optional enrichment**, while supporting materials live in their own secondary region. The CTA hierarchy was also corrected so **Generate first brief** is clearly primary and **Save draft** is visibly secondary.

This supports the Metis direction because Setup now behaves like a **live issue intake for briefing**, not onboarding, education, or enterprise workflow. A comms lead can identify the minimum fields required to create a defensible first draft quickly, while still adding context if it materially sharpens the brief.

Intentionally, this pass did **not** add onboarding steps, approval logic, routing, reminders, or workflow automation. It also avoided adding instructional copy that would turn the screen into a narrated demo.

The updated frame was validated in the browser and now reads as a clearer intake surface. The main remaining risk is that the shared operational snapshot still appears before the form and could be demoted further in a later polish pass.

## 2. Sources pass

The Sources screen was tightened so the **source list is the dominant work surface**. Source tiers are more clearly grouped, the weakest category is clarified in the UI as **Context signals**, and each source card now separates the main evidence body from metadata and linked-brief context. Supporting controls such as retagging, section-linking, and removal remain present but secondary.

This supports the Metis direction because Sources now feels like **evidence control**, which is central to a source-backed briefing workflow. The user can more quickly judge whether the brief rests on strong attributable material or weaker contextual inputs.

Intentionally, this pass did **not** turn Sources into a monitoring dashboard, live feed, or media intelligence product. It also avoided making linked sections dominate the screen over source strength.

The updated frame was validated in the browser and now reads much more like a product screen than an annotated concept board. The remaining weakness is that the page introduction could still be shortened slightly to feel more production-real.

## 3. Gaps pass

The Gaps screen was rebuilt so **unresolved gaps are the dominant content**. Each gap now shows the missing issue, why it matters, the affected brief section, the suggested stakeholder role, the drafted question, the current state, and a small set of owner-controlled actions. The layout clearly separates briefing risk from next-step support.

This supports the Metis direction because it makes Metis feel like more than a one-shot AI drafting tool. The screen now helps a user strengthen a live brief through structured clarification while still preserving human control.

Intentionally, this pass did **not** add assignees, reminders, approvals, stakeholder routing, or any other features that would pull the screen toward task management or workflow automation.

The updated frame was validated in the browser and now reads as structured clarification support. The main remaining risk is that the top action rail could be made even more compact in a later realism pass.

## 4. Brief refinements pass

The Brief workspace gained a stronger **readiness/status layer**, a visible **what changed since last version** layer, and a clearer **what blocks circulation** region, while keeping the document itself as the main surface. Section cards now separate briefing text, section state, editing actions, and linked evidence more clearly. The right rail also now includes a more credible short executive mode and manually captured internal signals.

This supports the Metis direction because the Brief workspace now answers the operational questions leadership teams need answered quickly: **what happened, what changed, what is unresolved, and whether the brief is safe to circulate**. That is much closer to Metis’ core artifact and value proposition.

Intentionally, this pass did **not** add a workflow engine, approvals, stakeholder routing, or any task-software framing. The brief remains the dominant artifact rather than being buried beneath status chrome.

The updated frame was validated in the browser and is materially closer to a real issue-briefing product. The main remaining risk is that the shared shell-level summary band still competes somewhat with the briefing-first hierarchy.

## 5. Browser validation / realism pass

A realism pass was completed in the browser across **Setup**, **Sources**, **Gaps**, and **Brief**. The prototype routes rendered correctly, the updated hierarchy system held across screens, and the experience now feels substantially less like a narrated concept prototype.

This supports the Metis direction because it demonstrates that the product is coherently expressed as a working internal tool, not just a set of isolated mockups. Type-check and production build were also completed successfully.

Intentionally, this pass did **not** broaden the information architecture, introduce new modules, or add workflow-heavy interactions during polish.

The main remaining realism risks are residual explanatory copy in some hero areas and the persistence of the shell-level operational snapshot across all screens, which can occasionally compete with the product-specific work surface beneath it.

## Category-drift check

The current design pass is **more tightly aligned with Metis** than the previous version. It now reads more clearly as an online issue-briefing workspace centered on a circulation-ready internal brief.

The main category-drift risk still present is the **shared operational snapshot band** at the top of every screen. If it grows further, it could start to pull the product toward a monitoring-led posture. Keeping that band secondary or selectively reducing it on deeper workflow screens would improve category discipline.
